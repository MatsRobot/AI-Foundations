## 🏗️ Repository Architecture

The project is divided into four functional modules:

### 1. 🔢 <small>[Neural Network Basics](https://github.com/MatsRobot/AI-Foundations/blob/main/Neural-Networks/)</small>
- **Mathematical Logic:** Mapping the flow of information through weights, biases, and activation functions.
- **The Data Pipeline:** Understanding how information flows through a model to build technical intuition.
- **Forward & Backward Passes:** Mastering the math behind the "black box."

### 2. 🛠️ <small>[AI-Tools](https://github.com/MatsRobot/AI-Foundations/blob/main/AI-Tools/)</small>
- **Custom Instructions:** Configuration files for steering LLMs toward embedded systems and robotics logic.
- **Prompt Engineering:** Structured templates for generating C++ hardware drivers and Python data scripts.
- **Workflow Automation:** Scripts and tools that bridge the gap between AI chat interfaces and local IDEs.

### 3. 🤖 <small>[AI-Assisted Development](https://github.com/MatsRobot/AI-Foundations/blob/main/AI-Development-Workflow/)</small>
- **Hardware Awareness:** Providing context for "Embedded C++" vs. standard C++ in AI prompts.
- **Debugging Best Practices:** Utilizing the "Rubber Duck" technique and iterative error analysis to resolve logical flaws.
- **Code Integrity:** Implementing strategies like the "Full File Mandate" and "Modular Prompting" to prevent AI code truncation.
- **Version Control:** Using Diff Tools to visually verify AI-generated changes before deployment.

### 4. 🐍 <small>[Python & IDE Setup](https://github.com/MatsRobot/AI-Foundations/blob/main/Python-Development-Environment/)</small>
- **VS Code vs. PyCharm:** Transitioning from "automatic" environments that hide the "plumbing" to lightweight, versatile professional tools.
- **Virtual Environments:** Managing specific library dependencies and absolute paths effectively to ensure application stability.
- **Polyglot Advantage:** Leveraging an environment that supports Python, C++, and other essential robotics languages in one place.

## 📐 The Learning Pipeline

1. **Theory:** Abstract the math of neural networks into logic gates.
2. **Tooling:** Configure specialized AI personas and prompts for robotics.
3. **AI-Augmentation:** Integrate AI as a debugging partner using a structured, hardware-aware approach.
4. **Environment Optimization:** Move logic into professional IDEs with proper environment linting.
5. **Verification:** Use "Rubber Duck" techniques and Diff tools to ensure code matches design intent.
